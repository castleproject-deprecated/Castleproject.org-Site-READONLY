LocalizationFilterAttribute

» Main Page » MonoRail » Documentation » MonoRail RC2 Documentation » Reference Manual » Attributes API Reference » LocalizationFilterAttribute

This is an special filter attribute. It is used to define from where MonoRail should read the localization information to find out the locale of the client.

For example, it can use the browser, or a cookie, an entry in the query string (or even in the session)

Constructors

LocalizationFilterAttribute()

Defines that you want to use a cookie named "locale", but if that fails it falls back to the client's browser locale.

LocalizationFilterAttribute(Castle.MonoRail.Framework.Filters.RequestStore store, System.String key)

Defines a new LocalizationFilter.

ParameterDescription
store Location where the localization parameter is stored.
key Name of the parameter in the store.

Properties

NameDescription
Key Key under which the locale value is stored.
Store Location where the locale value is to be stored.
FailOnError True if an exception is to be thrown when a specific culture appears to be incorrect (can't be created).
UseBrowser Use client browser defined languages as default.
FilterType Gets the filter implementation type
When Gets when to run the filter
ExecutionOrder Gets or sets the filter execution order. The lower the value, the higher the priority
