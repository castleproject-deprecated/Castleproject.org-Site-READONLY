db4o Facility

» Home » MicroKernel/Windsor » Documentation » Facilities » db4o Facility

The db4o Facility enables the usage of db4o O/R framework for your components.

Facility Summary
Uses ProxyNo
Requires configurationYes
Uses attributeNo
VersionBeta 1
Maturity levelMedium

Configuration Schema


<facilities>
  <facility id="db4o" 
            type="Castle.Facilities.Db4oIntegration.Db4oFacility, Castle.Facilities.Db4oIntegration">
    <container 
      id="db4o.container" 
      databaseFile="data/reloading.db4o" 
      hostName="HOSTNAME" 
      remotePort="51954" 
      user="username" 
      password="secret" 
      activationDepth="2" 
      updateDepth="2" 
      exceptionsOnNotStorable="true" 
      callConstructors="false"  />
  </facility>
</facilities>

Transaction support

This facility is aware of transactions, for more information check Automatic Transaction Management.
