Attributes

» Main Page » ActiveRecord » Documentation » ActiveRecord RC1 Documentation » Reference Manual » Attributes

This document explains briefly the attributes used by ActiveRecord and points to where to find more information about them.

The Used for schema? column indicates whether the value is used by the schema generation.

ActiveRecordAttribute (class level)

The following table lists the most common properties:

NameUsed for schema?Description
TableYesTable name associated with the type
Schema-schema name associated with the type
DiscriminatorColumn-Discriminator column for a table inheritance modeling
DiscriminatorType-column type (like string or integer) for the discriminator column
DiscriminatorValue-Value that represents the target class on the discriminator column
Lazy-Enable lazy loading for the type

The following table lists the other properties. They can be considered for advanced usage.

NameUsed for schema?Description
Proxy-Associates a proxy type with the target type
Where-SQL condition to retrieve objects
DynamicUpdate-From NHibernate documentation: Specifies that UPDATE SQL should be generated at runtime and contain only those columns whose values have changed
DynamicInsert-From NHibernate documentation: Specifies that INSERT SQL should be generated at runtime and contain only the columns whose values are not null
Persister-From NHibernate documentation: Specifies a custom IClassPersister
SelectBeforeUpdate-From NHibernate documentation: Specifies that NHibernate should never perform an SQL UPDATE unless it is certain that an object is actually modified. In certain cases (actually, only when a transient object has been associated with a new session using update()), this means that NHibernate will perform an extra SQL SELECT to determine if an UPDATE is actually required.
Polymorphism-From NHibernate documentation: Determines whether implicit or explicit query polymorphism is used. Value can be Implicit (default) or Explicit
Mutable-From NHibernate documentation: Specifies that instances of the class are (not) mutable.
BatchSize-From NHibernate documentation: Specify a "batch size" for fetching instances of this class by identifier. Defaults to 1
Locking-From NHibernate documentation: Determines the optimistic locking strategy. Value can be None, Version, Dirty or All. Defaults to Version.
UseAutoImport-From NHibernate documentation: The auto-import attribute lets us use unqualified class names in the query language, by default. The assembly and namespace attributes specify the assembly where persistent classes are located and the namespace they are declared in.
